# K.AI Labs backend

Node.js + TypeScript + Express + Prisma (PostgreSQL). Provides the JSON API used by the Chrome
extension and a minimal server-rendered web UI (login, signup, dashboard, extension connect).

## Local development

1. Have a PostgreSQL database available (local install, Docker, or a free hosted one e.g. Supabase).
2. `cp .env.example .env` and fill in `DATABASE_URL`, `JWT_SECRET`, `MASTER_KEY` (see comments in
   the file for how to generate secrets).
3. From the repo root: `npm install`
4. `npx prisma migrate dev` (run from `backend/`) — creates tables.
5. `npm run dev` (from `backend/`, or `npm run dev:backend` from root) — starts on `http://localhost:3000`.
6. Visit `http://localhost:3000`, sign up, add an AI API key, then open `/extension/connect` from
   the same browser you'll load the unpacked extension in.

## Deploying to Render (so non-technical users can use it online)

1. Push this repo to GitHub.
2. In Render, choose "New +" -> "Blueprint" and point it at the repo. Render will read
   `backend/render.yaml` and provision the web service + a free Postgres database automatically.
3. Set the `APP_BASE_URL` env var on the web service to the public URL Render assigns
   (e.g. `https://kai-labs-backend.onrender.com`) — Render will prompt for this since it's marked
   `sync: false`.
4. Deploy. Render runs `prisma migrate deploy` as part of the build, so the database schema is
   created automatically on first deploy.
5. Once live, point the extension at the deployed URL (see `extension/README.md`).

`JWT_SECRET` and `MASTER_KEY` are auto-generated by Render's blueprint (`generateValue: true`) —
no manual secret handling needed for a first deploy.
